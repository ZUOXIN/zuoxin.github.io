HOW TO WRITE

From the very beginning， 我需要纸和笔。

我要的是什么？

  • 自动同步（备份、发布、共享）。
  • 不受编辑器限制，可以在linux命令行下发布。

Farbox

  • 收费。
  • 托管 markdown 文件。
  • 域名绑定。
  • 支持 Dropbox。
  • 支持属性配置，支持YAML。
  • 可选择模板。

Farbox Editor

  • 自动同步到 Farbox 服务器，但不能同步到Dropbox。
  • 可预览 markdown。
  • 可作为本地编辑器，并支持导出pdf。
  • 拖拽导入图片，并可调整尺寸。

postach.io

  • 收费。
  • 直接在 Evernote 中添加 publish 的 tag 发布。
  • 域名绑定。
  • 可直接选择模板。

pancake.io

  • 托管 html 和 markdown 文件。
  • 托管 jpg, png, gif, ico, psd, ai, ps, pdf, docx, xlsx, pptx 等文件。
  • 支持 Dropbox 。
  • 支持 Git 方式同步。
  • 域名绑定。
  • 可直接选择模板。

Jekyll

  • 自动生成静态网页，不负责托管。
  • 支持 html 和 markdown 文件。
  • 所生成的 html 文件采用标签语言作为模板，需要调教。
  • 每篇文章必须有 YAML 文件头。
  • 有开发服务器可本地预览。

Github Pages

  • 托管 Jekyll，上传 html/md文件 后自动生成页面显示。
  • 托管 Hexo，只有生成后的静态页面，不保存 md 文件。
  • 采用 Github 同步。
  • 域名绑定。

Hexo

  • 命令行生成静态网页，不负责托管。
  • 命令行发布到 GiuhubPages。
  • 支持GFM (GitHub Flavored Markdown)。
  • 每篇文章必须有 YAML 文件头。
  • 有开发服务器可本地预览。

Evernote

  • 自动同步。
  • 拖拽导入文件。

Emacs & org-mode

  • 可导出为 markdown 文件。
  • 可导出为 html 文件。
  • 可导出为 pdf 文件。
  • 可调教 evernote-mode 。

Atom

  • Github自己的编辑器。
  • 默认装有Markdown Preview。
  • 默认支持 GFM (GitHub Flavored Markdown)。

Evernote for Sublime Text

  • Sublime Text 插件
  • 手动同步
  • 双向同步（但修改表格会导致变html）

EVND (Ever Notedown)

  • Atom 插件
  • 本地保存，双向同步

markable.in

  • 文件保存在网站上
  • 手动同步
  • md/html互相导入导出
  • 在Evernote中把token删掉后就再也用不了了（哭

Enjoy!
